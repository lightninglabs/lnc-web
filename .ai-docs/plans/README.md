# `.ai-docs/plans/`

Plans capture the approach used to instruct an AI model for a change.

- Plans are **authored by humans**
- Plans are **committed**
- Plans are **not generated** or synced to any harness paths

## Naming

Use: `<YYYY-MM-DD>_<short_description>.md`

## Example

```
.ai-docs/plans/
  2026-01-22_ai_files_sync.md
```

Example plan contents:

- Goal / scope
- Constraints and assumptions
- Implementation steps
- Test plan
